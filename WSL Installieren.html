Schritt 1: WSL aktivieren

  1. Öffnen Sie eine PowerShell-Konsole als Administrator:

    • Klicken Sie auf das Windows-Symbol, suchen Sie nach PowerShell, klicken Sie mit der rechten Maustaste und wählen Sie Als Administrator ausführen.

  2. Aktivieren Sie WSL mit diesem Befehl:

    wsl --install

    Was passiert dabei?

    • WSL wird installiert.
    • Die Virtual Machine Platform wird aktiviert.
    • Die neueste Ubuntu-Version wird automatisch heruntergeladen und installiert.

  3. Neustart erforderlich:
    Wenn Sie dazu aufgefordert werden, starten Sie Ihren Computer neu.

Schritt 2: Optionale manuelle Installation

Falls Sie mehr Kontrolle benötigen, z. B. eine spezifische Ubuntu-Version installieren möchten oder WSL 1 bevorzugen, können Sie die Schritte anpassen.

WSL-Funktion und Virtual Machine Platform separat aktivieren

Falls wsl --install nicht funktioniert, aktivieren Sie die Komponenten manuell:

dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart
dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart

Danach Neustart durchführen.

Standardversion auf WSL 2 setzen

wsl --set-default-version 2

Ubuntu-Version manuell installieren

Um eine spezifische Ubuntu-Version zu installieren (z. B. 20.04 oder 22.04), nutzen Sie:

wsl --install -d Ubuntu-20.04

Sie können verfügbare Distributionen auflisten:

wsl --list --online

Schritt 3: Ubuntu starten und konfigurieren

  1. Sobald die Installation abgeschlossen ist, starten Sie die Ubuntu-Distribution:

    wsl

    oder

    wsl -d Ubuntu-20.04

  2. Legen Sie Ihren Linux-Benutzernamen und Ihr Passwort fest.

  3. Aktualisieren Sie das System:

    sudo apt update && sudo apt upgrade -y

Nützliche Befehle zur Verwaltung von WSL

  • Installierte Distributionen anzeigen:

    wsl --list --verbose

  • Standard-Distribution ändern:

    wsl --set-default <Distribution-Name>

  • Distribution deinstallieren:

    wsl --unregister <Distribution-Name>

  • WSL aktualisieren:

    wsl --update

Vorteile der Installation über die Kommandozeile

  • Schneller und flexibler als der Microsoft Store.
  • Ermöglicht die Installation spezifischer Distributionen oder Versionen.
  • Ideal für skriptgesteuerte Setups oder automatisierte Installationen.
